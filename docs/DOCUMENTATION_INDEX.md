## üéì Key Concepts Explained

### Architecture Layers
1. **Client Layer**: Browser with HTML/CSS/JavaScript
2. **Web Server Layer**: Flask application with route handlers
3. **Business Logic Layer**: Content management, file handling
4. **Data Access Layer**: SQLite database with queries
5. **File Storage Layer**: Static files (PDFs, images, audio)

### Content Types
- **Books**: Traditional PDFs with optional audio
- **Manga**: Series with multiple chapters (sequential PDFs)

### User Workflows
1. **Reader**: Register ‚Üí Login ‚Üí Browse ‚Üí Read ‚Üí Watchlist
2. **Publisher**: Register ‚Üí Request Role ‚Üí Approved ‚Üí Upload ‚Üí Manage
3. **Admin**: Login (pre-existing) ‚Üí Manage Users/Team/AI Cache

### Data Flow Pattern
- User Action ‚Üí Route Handler ‚Üí Validation ‚Üí File Processing (if needed) ‚Üí Database Update ‚Üí Response ‚Üí UI Update
